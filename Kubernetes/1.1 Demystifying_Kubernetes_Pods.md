### ðŸ”¹ 1. What Is a Pod?

A **Pod** is the *smallest deployable unit* in Kubernetes.
It represents **one or more containers** that share:

- The same network namespace (same IP & port space)
- Shared storage (optional)
- Lifecycle (they start, stop, restart together)

Think of a Pod as a **wrapper around one or more containers** that must run together.

### ðŸ”¹ 2. YAML Breakdown

### **`myfirstpod.yml`**

```yaml
apiVersion: v1
kind: Pod
metadata:
  name: nnappone
  labels:
    app: nnappone
spec:
  containers:
    - name: networknuts-app
      image: lovelearnlinux/webserver:v1

```

---

### ðŸ§  **Explanation (Line-by-Line)**

| Section | Field | Purpose |
| --- | --- | --- |
| **apiVersion** | `v1` | Refers to core Kubernetes API group used for Pods. |
| **kind** | `Pod` | Defines what object you are creating. |
| [**metadata.name**](http://metadata.name/) | `nnappone` | Unique pod name in the namespace. |
| **metadata.labels** | `app: nnappone` | Used for grouping and selectors (for Services, ReplicaSets, etc.). |
| **spec.containers** | `[ ... ]` | List of containers to run inside the Pod. |
| [**containers.name**](http://containers.name/) | `networknuts-app` | Logical name of the container (for logs, exec, etc.). |
| **containers.image** | `lovelearnlinux/webserver:v1` | Docker image from registry. Kubernetes will pull and run it. |

### ðŸ§© **Concepts Covered**

- Pod is atomic â€” it cannot be split.
- Each Pod gets its **own IP address**.
- Containers inside a Pod **share localhost networking**.
- Pods are **ephemeral** â€” if deleted, they do not self-recover (unless managed by a controller).
